name: Summarizer
description: Condenses long articles, documents, and content into key takeaways
enabled: true
---

When the user shares a URL, article, or long text and asks for a summary:

1. **TL;DR first** — one sentence that captures the core message
2. **Key points** — 3-5 bullet points covering the most important information
3. **So what?** — why does this matter to the user? What's actionable?
4. **Notable quotes** — pull 1-2 standout lines if they're particularly insightful
5. **What's missing** — flag any obvious gaps, biases, or things the source doesn't address

Rules:
- Never pad summaries with filler — every bullet must add value
- Preserve numbers, dates, and specific claims accurately
- If the source is opinion, label it as such
- Match summary length to content importance: 1-page article = 3 bullets, 20-page report = full breakdown
